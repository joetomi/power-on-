# Router Status

مشروع Next.js لمراقبة اتصال الراوتر وتسجيل أوقات الانقطاع والعودة في ملف JSON دائم على Vercel Blob، دون قاعدة بيانات.

## التشغيل محليًا

```bash
npm install
copy .env.example .env.local
npm run dev
```

ضع رمز Vercel Blob في `.env.local`:

```env
BLOB_READ_WRITE_TOKEN=your_token_here
```

ثم افتح `http://localhost:3000`.

## إعداد Vercel Blob

1. افتح مشروعك في Vercel.
2. انتقل إلى **Storage** ثم أنشئ **Blob Store** جديدًا.
3. اختر **Private** كنوع الوصول.
4. اربط الـ Blob Store بالمشروع وبيئات Production وPreview المطلوبة.
5. يضيف Vercel المتغير `BLOB_READ_WRITE_TOKEN` تلقائيًا عند ربط التخزين بالمشروع. إذا لم يظهر، انسخ الرمز من إعدادات Blob وأضفه في **Settings → Environment Variables**.
6. أعد نشر المشروع بعد إضافة المتغير.

لا تضع قيمة الرمز في Git أو في أي متغير يبدأ بـ `NEXT_PUBLIC_`.

## ملف السجل

ينشئ المشروع ملفًا خاصًا باسم `events.json` تلقائيًا عند أول استخدام إذا لم يكن موجودًا. يحتوي الملف على:

- الحالة المؤكدة الأخيرة.
- عدادات النجاحات والإخفاقات المتتالية.
- أحداث `online` و`offline` مع التاريخ بصيغة ISO والقيمة الرقمية `timestampMs`.

تمنع الكتابة المشروطة باستخدام ETag تضارب عمليات الفحص المتزامنة، كما لا يكتب المشروع الملف عندما تكون الحالة والعدادات ثابتة.

## آلية تأكيد الحالة

- يثبت الانقطاع بعد 3 عمليات فحص فاشلة متتالية.
- تثبت عودة الاتصال بعد عمليتي فحص ناجحتين متتاليتين.
- لا يسجل المشروع حدثًا إذا لم تتغير الحالة المؤكدة.
- تحدد أول حالة مؤكدة خط الأساس فقط، ولا تسجل انقطاعًا وهميًا عند بدء المشروع لأول مرة.
- يستمر الفحص كل 5 ثوانٍ أثناء فتح الصفحة الرئيسية أو صفحة التقارير.

## التقارير

افتح المسار التالي مباشرة:

```text
/reports
```

تعرض الصفحة التواريخ بتوقيت `Africa/Tripoli`، ومدد الانقطاعات، والإحصاءات الإجمالية. يمكن تنزيل السجل بزر **تنزيل CSV**.

لحفظ التقرير بصيغة PDF، اضغط **طباعة التقرير** ثم اختر **Save as PDF / حفظ كملف PDF** من نافذة الطباعة.

## واجهات API

- `GET /api/status`: يفحص الراوتر ويحدّث الحالة والسجل عند الحاجة.
- `GET /api/history`: يعيد الأحداث من الأحدث إلى الأقدم.

## التحقق والبناء

```bash
npm run typecheck
npm run build
```

المشروع جاهز للنشر المباشر على Vercel باستخدام إعدادات Next.js الافتراضية.
